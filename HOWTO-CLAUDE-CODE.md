# 🚀 HOWTO — Build Golazo Engine with Claude Code

دليل مختصر لتشغيل الملفات الخمسة في Claude Code وبناء الخدمة كاملة ورفعها.

---

## 0. التحضير
1. أنشئ مجلد فارغ جديد للمشروع، مثلاً `golazo-engine/`، وافتحه في VS Code.
2. انسخ الملفات الخمسة (`agent.md`, `planning.md`, `tasks.md`, `rules.md`, `design.md`)
   + هذا الملف داخل المجلد (يفضّل في جذر المشروع).
3. (اختياري لكن مفيد) ضع نسخة من `golazo_studio.html` في المجلد — هو **المصدر البصري**
   الذي تُنقل منه دوال الإطار والزخارف حرفياً.

## 1. أمر البداية في Claude Code
افتح Claude Code في مجلد المشروع، والصق هذا كأول رسالة:

```
اقرأ agent.md و planning.md و rules.md و design.md و tasks.md بالكامل أولاً.
هذا مشروع Node + Express + @resvg/resvg-js لتوليد بطاقات PNG.
ابدأ بتنفيذ Phase 1 في tasks.md فقط، وتوقّف بعدها لأراجع.
القاعدة الأهم: resvg لا يدعم foreignObject — كل نص عربي يُرسم كـ <text> أصلي
عبر دوال arText/arBox/arBlock الموضّحة في design.md. لا تستخدم foreignObject إطلاقاً.
```

ثم بعد كل Phase راجع وقل له "تابع Phase التالي".

## 2. الترتيب الموصى به للتنفيذ
- **Phase 1** — setup + تحميل الخطوط. تأكد بنفسك أن `file fonts/*.ttf` يقول TrueType.
- **Phase 2** — نواة الرندر (دوال النص + الإطار + render.js). هذا أهم وأصعب جزء.
- **Phase 3** — القوالب الأربعة (انسخ أجسامها من design.md حرفياً).
- **Phase 4** — السيرفر + اختبار محلي. شغّل `node test_cards.js` وافتح صور `/tmp/card_*.png`
  وتأكد بصرياً: العربي مشكّل، الاتجاه RTL صحيح، لا نص مفقود، الزخارف موجودة.
- **Phase 5** — رفع على Railway (GitHub repo جديد منفصل عن ريبو التطبيق).
- **Phase 6** — ربط n8n (برومبت DeepSeek الجديد + HTTP node + Telegram sendPhoto).

## 3. نقطة التحقق الحرجة قبل الرفع
لا ترفع حتى تشوف بعينك بطاقة `breaking` تطلع صحيحة محلياً مع:
- العنوان ملفوف على أكثر من سطر بشكل سليم ومتمركز.
- التفاصيل (متعددة الأسطر) ظاهرة.
- إطار Golazo كامل (الزخارف + الهاشتاق + فوتر "الكرة بالأرقام · @golazo.arabic").

## 4. الرفع على Railway (تلخيص)
1. أنشئ GitHub repo جديد (مثلاً `golazo-engine`) — **منفصل** عن ريبو التطبيق.
2. تأكد لا يوجد أي مفتاح سري (هذه الخدمة لا تحتاج مفاتيح). ادفع كل الملفات **مع مجلد fonts/**.
3. Railway → New → Deploy from GitHub repo → اختر الريبو.
4. انتظر Active، ثم Settings → Networking → فعّل Public Domain واحفظ الرابط.
5. افتح `https://<domain>/health` — يجب أن يرجّع القوالب الأربعة. هذا دليل النجاح.

## 5. ملاحظة عن المرجع الجاهز
يوجد تطبيق مرجعي عامل بالكامل (نفس هذه المواصفات) في ملف `golazo-engine.zip`
الذي حصلت عليه سابقاً. يمكنك استخدامه للمقارنة أو كنقطة انطلاق إذا أردت تسريع العمل،
لكن هذه الملفات الخمسة تكفي لبناء كل شيء من الصفر.

## 6. خارج النطاق الآن (مراحل لاحقة)
قوالب البيانات (fixtures/results/top10 من api-football)، القوالب الهجينة
(statshock/comparison)، نشر Buffer + بوابة الموافقة، وقاعدة PostgreSQL — كلها
مراحل تالية منفصلة، لا تبنِها الآن. ركّز على فئة الأخبار حتى تكتمل وتنشر بطاقات فعلية.
